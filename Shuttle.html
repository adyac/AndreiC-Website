1. Choix des Hyperparamètres

Ce projet implémente un réseau de neurones de type auto-encodeur dédié à la détection d'anomalies sur le dataset Shuttle. Le modèle apprend à reconstruire exclusivement les données normales afin d'identifier les déviations lors de la phase de test.

La sélection des paramètres repose sur une analyse statistique des données pour garantir une compression optimale.

Paramètres Retenus
  • Espace Latent (k = 6) : Basé sur l'analyse des valeurs propres — on observe 6 valeurs significatives
  • Taux d'apprentissage : 0,0015 — assure une stabilisation fluide de la perte
  • Optimiseur : Adam — convergence efficace
  • Epochs : 80 — le plateau de convergence est atteint vers la 76ème époque

Analyse des Valeurs Propres

2. Architecture du Réseau

Le réseau adopte une structure symétrique en "sablier" pour extraire les caractéristiques essentielles des signaux.

Spécifications de l'Architecture
  • Dimensions des entrées/sorties : 1×9 → 1×9
  • Encodeur : Trois couches linéaires progressives (9 → 8 → 7 → 6)
  • Activations : ReLU après chaque couche linéaire
  • Espace Latent : Dimension compressée de 6
  • Décodeur : Structure miroir de l'encodeur (6 → 7 → 8 → 9)
  • Sortie finale : Aucune activation — reconstruction fidèle des valeurs réelles

L'utilisation de couches ReLU après chaque couche linéaire introduit la non-linéarité nécessaire pour capturer les relations complexes dans les données. L'absence d'activation en sortie permet une reconstruction non contrainte des valeurs continues.

3. Stratégie d'Entraînement

L'entraînement est optimisé pour traiter les données normales tout en ignorant les anomalies.

Paramètres d'Entraînement
  • Batch Size : 256 — maximise la vitesse de calcul tout en évitant une généralisation excessive
  • Entraînement : Exclusivement sur les données de classe normale (classe 1)
  • Fonction de perte : Mean Squared Error (MSE)
Prétraitement & Normalisation
  • Prétraitement : Élimination des données invalides et séparation des étiquettes pour ne garder que les 9 dimensions des capteurs
  • Normalisation : Calculée uniquement sur les données d'entraînement valides (classe 1)
  • Test : Normalisé avec les moyennes et déviations standard du set d'entraînement pour éviter que les anomalies ne biaisent l'échelle

4. Détermination du Seuil Optimal

La classification finale repose sur le calcul de l'erreur de reconstruction (MSE). Un seuil est déterminé pour séparer le "sain" de l'"anormal".

Méthode de Détermination
  • Objectif : Recherche du compromis idéal minimisant les Faux Négatifs (FN) et les Faux Positifs (FP)
  • Analyse visuelle : Utilisation d'histogrammes de pertes pour identifier la séparation entre les deux distributions
  • Évaluation : Validation par la courbe de la F-measure (F1-score) en fonction du seuil

6. Conclusion

Résumé du Projet

Ce projet démontre qu'un auto-encodeur léger avec un espace latent de dimension 6 suffit pour détecter efficacement les anomalies dans le dataset Shuttle. L'approche de reconstruction — entraîner le modèle uniquement sur des données normales puis seuiller l'erreur MSE — s'avère robuste et interprétable.

Points Clés

  • F-score de 0,9912 — performance quasi-parfaite sur ce jeu de données
  • Accuracy de 98,62% — généralisation excellente
  • Architecture symétrique minimaliste — 9 → 6 → 9 avec ReLU
  • Convergence stable atteinte dès la 76ème époque sur 80
  • Seuil optimal déterminé par analyse F-measure, minimisant FP et FN

Améliorations Futures

  • Variational Autoencoder (VAE) : Modélisation probabiliste de l'espace latent pour une meilleure généralisation
  • Recherche de seuil automatique : Optimisation du seuil par validation croisée
  • Autres datasets : Tester la robustesse sur KDD Cup, MNIST-Anomaly, etc.
  • LSTM Autoencoder : Exploiter la dimension temporelle des données de capteurs
